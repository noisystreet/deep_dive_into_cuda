PTXAS 分析：PTX→SASS 汇编器
===============================

   ptxas = PTX Assembler，是 CUDA 工具链中负责将 PTX 虚拟汇编代码 编译为
   SASS（Streaming ASSembler，GPU 实际机器码）的关键组件

   分析基于 CUDA 13.1 (build 37061995)

--------------

-------------

   大小: 41,357,392 字节 (≈ 40 MB)

         dynamically linked, stripped

         Build cuda_13.1.r13.1/compiler.37061995_0

   libpthread.so.0     ← 多线程（Finalizer 阶段使用）

-----------

ptxas 采用 **两阶段 (Two-Phase)** 架构：

^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

=========================================== ================
阶段                                        说明
=========================================== ================

Finalizer 阶段 (多线程)
^^^^^^^^^^^^^^^^^^^^^^^

::

   [Finalizer] spawning %d threads with dyn scheduling    ← 动态线程调度
   [Finalizer] spawning %d threads with static scheduling ← 静态线程调度
   [Finalizer] fastpath optimization applied for off-target %u -> %u

Finalizer
使用线程池进行并行处理，可选择动态或静态调度策略。\ ``--split-compile <N>``
控制并发线程数。

指令调度与延迟隐藏
^^^^^^^^^^^^^^^^^^

::

   ScheduleInstructions     ← 指令调度 Pass
   PostSchedule             ← 后调度 Pass
   LDG latency hiding       ← 全局加载延迟隐藏
   LDS latency hiding       ← 共享内存加载延迟隐藏
   Antidep latency hiding   ← 反依赖延迟隐藏
   Deferred-barrier latency hiding ← 延迟 barrier 隐藏

ptxas 内置了专门的\ **延迟隐藏分析器**\ ，通过对 warp
调度器行为的建模，插入无关指令来隐藏内存加载的延迟。

循环优化
^^^^^^^^

ptxas 包含丰富的循环展开逻辑，识别 18 种不同的”不展开原因”：

::

   0x80000001 - Not unrolled: Irregular loop           ← 不规则循环
   0x80000002 - Not unrolled: Irregular loop           
   0x80000007 - Not unrolled: performance              ← 性能原因（展开无益）
   0x8000000d - Not unrolled: Unsupported loop type    ← 不支持循环类型
   0x8000000e - Not unrolled: Unsupported loop exit    ← 不支持退出条件
   0x80000010 - Not unrolled: Unsupported loop index   ← 不支持索引变量

寄存器分配
~~~~~~~~~~~~~~

寄存器使用可通过多个维度控制：

1. **``-maxrregcount <N>``**: 硬性限制每个 kernel 最大寄存器数
2. **``--register-usage-level <0..10>``**: 基于激进度的软控制

   -  **0**: 最小化寄存器使用（可能产生大量 spill）
   -  **5**: 默认平衡
   -  **10**: 最大化性能，可能使用更多寄存器

3. **``--device-function-maxrregcount <N>``**: device 函数单独控制

当寄存器不足时，变量被 **spill** 到 **local memory**\ （即栈内存，通过
``--warn-spills`` 可检测）。

SASS 指令编码
~~~~~~~~~~~~~~~~~

每一条 SASS 指令都被编码为 **64 位 (2 个 32-bit word)** 格式：

::

   Word 0 (bits 63-32):     Word 1 (bits 31-0)
   ┌─────────────────────┐ ┌─────────────────────┐
   │     opcode + fields  │ │     operands        │
   └─────────────────────┘ └─────────────────────┘

cuobjdump 的 ``-sass`` 输出中，每条指令对应一个 hex 值：

.. code:: text

   /*0090*/  IMAD R6, R0, c[0x0][0x178], R5 ;  ← 汇编表示
             0x00005c0006067625              ← 二进制编码 (小端)

编码字段分解（推测）：

========== ==== ========================
位域       宽度 说明
========== ==== ========================
bits 0-9   10   目标寄存器地址
bits 10-19 10   源操作数 A 地址
bits 20-29 10   源操作数 B 地址/立即数
bits 30-39 10   源操作数 C/常量偏移
bits 40-46 7    opcode (主指令码)
bits 47-63 17   修饰符、谓词、条件标志等
========== ==== ========================

PTX → SASS 映射示例
~~~~~~~~~~~~~~~~~~~~~~~

分析同一 kernel 的 PTX 与 SASS 差异：

.. code:: text

   ; PTX (虚拟 ISA)
   mad.lo.s32 %r1, %r3, %r4, %r5;   idx = blockIdx * blockDim + threadIdx
   setp.ge.s32 %p1, %r1, %r2;       idx >= n ?
   @%p1 bra $L__BB0_2;              条件跳转
   ld.global.f32 %f1, [%rd8];       加载 b[idx]
   add.f32 %f3, %f2, %f1;           a[idx] + b[idx]

   ; SASS (实际机器码)
   IMAD R6, R0, c[0x0][0x178], R5;  idx = blockIdx * blockDim + threadIdx
   ISETP.GE.AND P0, PT, R6, c[0x0][0x170], PT;  idx >= n ?
   @P0 BRA 0x100;                    条件跳转 (谓词执行)
   LDG.E R4, [R4.64];               加载 b[idx] (带缓存提示)
   FADD R9, R4, R3;                 a[idx] + b[idx]

**关键差异**:

+-----------------------+-----------------------+-----------------------+
| 对比项                | PTX                   | SASS                  |
+=======================+=======================+=======================+
| **寄存器**            | 虚拟 (``%r1``,        | 物理 (``R0-R9``,      |
|                       | ``%f1``, ``%rd1``)    | ``P0``)               |
+-----------------------+-----------------------+-----------------------+
| **常量参数**          | 隐式 (``%ctaid.x``)   | 显式                  |
|                       |                       | (``c[0x0][0x178]`` =  |
|                       |                       | constant bank)        |
+-----------------------+-----------------------+-----------------------+
| **内存访问**          | ``ld.global.f32``     | ``LDG.E`` (带缓存提示 |
|                       |                       | ``.E``)               |
+-----------------------+-----------------------+-----------------------+
| **地址计算**          | ``cvta.to             | ``IMAD.WIDE``         |
|                       | .global + mul + add`` | (融合乘加寻址)        |
+-----------------------+-----------------------+-----------------------+
| **控制流**            | ``bra`` 标签          | ``BRA`` 绝对偏移地址  |
+-----------------------+-----------------------+-----------------------+
| **类型信息**          | 在指令中 (``f32``,    | 操作码隐含 (IMAD 总是 |
|                       | ``s32``)              | 32-bit)               |
+-----------------------+-----------------------+-----------------------+

PTX ISA 兼容性管理
---------------------

ptxas 内部维护了精确的 PTX ISA 版本兼容性表：

::

   Feature '%s' requires PTX ISA .version %s or later    ← 特性版本要求
   Instruction '%s%s%s%s' requires PTX ISA .version %d.%d or later ← 指令版本要求
   Instruction '%s%s%s%s' requires .target sm_%d or higher ← 架构要求
   Instruction or declaration violates .target texmode_%s setting ← 纹理模式
   Instruction '%s' without '.sync' is deprecated since PTX ISA version 6.0

支持的计算架构
~~~~~~~~~~~~~~~~~~

::

   PTX .target 架构范围 (compute_*):
     compute_75, compute_80, compute_86, compute_87,
     compute_100, compute_100a, compute_100f,
     compute_103, compute_103a, compute_103f,
     compute_110, compute_110a, compute_110f,
     compute_120, compute_120a, compute_120f,
     compute_121, compute_121a, compute_121f

   SASS 目标架构范围 (sm_*):
     sm_20, sm_21, sm_30, ...
     sm_75, sm_86, sm_89, ...
     sm_100, sm_100a, sm_100f,
     sm_103, sm_103a, sm_103f,
     sm_110, sm_110a, sm_110f,
     sm_120, sm_120a, sm_120f,
     sm_121, sm_121a, sm_121f

后缀含义： - ``a`` = 针对该代架构的特化变体 (automotive?) - ``f`` =
针对该代架构的另一个变体 (full feature?)

PTX → SASS 编译规则
~~~~~~~~~~~~~~~~~~~~~~~

::

   PTX for .target sm_XY can be compiled to all GPU targets sm_MN, sm_MNa, sm_MNf 
   where MN >= XY.

   PTX for .target sm_XYf can be compiled to GPU targets sm_XZ, sm_XZf, sm_XZa 
   where Z >= Y and sm_XY and sm_XZ belong in same family.

   PTX with .target sm_XYa can only be compiled to GPU target sm_XYa.

这意味着 PTX 只向前兼容——为 sm_75 编译的 PTX 可以被 ptxas 编译为 sm_89
的 SASS。

Tensor Core / MMA 支持
-------------------------

ptxas 包含了丰富的 Tensor Core 处理逻辑，用于支持 NVIDIA
的矩阵乘加指令：

::

   wgmma.mma_async pipeline                   ← wgmma 流水线
   wmma (Warp Matrix Multiply Accumulate)      ← Tensor Core 矩阵乘加

   性能警告：
     Potential Performance Loss: wgmma.mma_async instructions are serialized due to
       - ill formed pipeline stage
       - insufficient register resources
       - non wgmma instructions defining accumulator/input registers between pipeline stages
       - wgmma pipeline crossing function boundary

地址空间变体：

::

   .global        ← 全局内存加载
   .shared        ← 共享内存加载
   .descriptor    ← TMA 描述符加载

Sanitizer 集成
-----------------

ptxas 支持直接生成带 sanitizer 插桩的代码（\ ``--sanitize`` 选项）：

memcheck 检测越界访问）。
